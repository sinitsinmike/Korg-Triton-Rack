## Confirmed Findings

### 1. Pointer table at 0x00000068

Status: CONFIRMED

Evidence:

- Detected by pointer scanner
- Verified manually in Ghidra
- Contains contiguous 32-bit pointer values

Conclusion:

This is the first confirmed pointer table in the ROM.

---

### 2. Structured region beginning near 0x003A9000

Status: CONFIRMED

Evidence:

- Significant entropy drop
- Large amount of repeating records
- Structured appearance in Ghidra

Conclusion:

This region is very unlikely to contain executable code.
It probably contains metadata or indexed records.

---

### 3. Pointer cluster at 0x0006543C

Status: UNDER INVESTIGATION

Contains 24 pointers:

0x001D5108
0x0023F1F6
...
0x007E9C6A

Observations:

- Addresses increase almost monotonically.
- Targets appear to contain structured data.
- No evidence that these are function entry points.

Current hypothesis:

Large resource index (waves, samples, banks, or similar ROM objects).

---

### 4. Candidate executable region

Range:

0x00100000+

Method:

Entropy-based detector.

Result:

Ghidra decoded a few valid SH-2 instructions,
but continuous disassembly failed.

Conclusion:

Entropy alone is not sufficient for executable code detection.

Future code detector should also include:

- pointer density
- branch density
- instruction validity
- jump table detection
