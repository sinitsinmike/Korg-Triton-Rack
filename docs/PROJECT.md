# Korg Triton Rack Reverse Engineering

## Mission

Understand the complete firmware architecture of the Korg Triton Rack.

## Goals

- Identify CPU architecture
- Reconstruct memory map
- Understand boot process
- Document flash layout
- Locate hardware initialization
- Understand System Error Code 9
- Produce complete documentation

## Rules

1. No assumptions without evidence.
2. Every script answers one question.
3. Every discovery is documented.
4. Never overwrite original ROM dumps.
5. Every generated file must be reproducible.

## Repository Philosophy

This repository is both:

- research notebook
- reverse engineering toolkit
